# Cortex Commands

This file separates installed slash commands from skill trigger phrases. A slash command is only real when an adapter writes a command file.

## Cortex-Owned Claude Commands

Generated by [`scripts/sync-claude-commands.sh`](scripts/sync-claude-commands.sh):

| Command | What It Does |
| --- | --- |
| `/handoff` | Runs the `handoff` closeout workflow. |
| `/closeout` | Alias for the same closeout workflow. |
| `/asbuilt` | Runs `asbuilt` in `derive`, `verify`, or `conform` mode. |
| `/studio-audit` | Runs the final senior-designer audit orchestrator. |

## `/asbuilt` Modes

| Mode | What It Does |
| --- | --- |
| `derive` | Recover a design-system package from an existing codebase. Read-only on the target repo. |
| `verify` | Audit an existing asbuilt package and decide whether agents can rely on it. Read-only. |
| `conform` | Apply an existing package back into code on a local scratch branch. This can modify code. |

## Skill Triggers That Are Not Cortex-Owned Commands

These strings appear in skills or examples, but Cortex does not install command files for them:

| Trigger | Source |
| --- | --- |
| `/fable-codex` | `agent-workflows/fable-codex` describes this as opt-in session mode. |
| `/loop` | `agent-workflows/fable-prompting` ships a drop-in loop template, not a Cortex command adapter. |
| `/interface-craft` | `design/craft/interface-craft` uses this as an invocation pattern. |
| `/caveman` | `engineering/caveman` lists this as user trigger language. |
| `/rams` | `design/tools/rams` documents the external Rams command, but Cortex does not generate it. |
| `/setup-matt-pocock-skills` | Several engineering skills mention this setup trigger, but Cortex does not generate the slash file. |

## Rule

If a command should become real, add it to `scripts/sync-claude-commands.sh` and document it here. If it is only trigger language, keep it in the skill and do not call it a Cortex-owned command.

# imagery

this file describes image, icon, and diagram treatment.

## product surfaces

authenticated product UI should not rely on atmospheric imagery.

use images only when they show:

- user-provided content.
- product previews.
- diagrams that explain structure.
- thumbnails attached to records.

## icons

- use one icon family.
- keep icon stroke weight consistent.
- icon-only controls need labels and tooltips.
- icons should support commands, navigation, and status, not replace all text.

## diagrams

diagrams should be simple enough to inspect quickly.

use diagrams for:

- workflow states.
- architecture handoff.
- data relationships.

open decisions:

- approved icon library.
- thumbnail crop ratios.
- chart and data visualization style.

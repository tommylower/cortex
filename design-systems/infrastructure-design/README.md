# infrastructure-design

`infrastructure-design` is an extracted technical marketing-site design system for institutional, dark-mode product surfaces.

It is built around ink and paper contrast, precise hairlines, square geometry, sparse signal color, mono supporting type, and diagrammatic motion.

## what it contains

- `SKILL.md` — operating brief, design thesis, core rules, and reference map
- `references/tokens.md` — semantic color, type, spacing, radius, and shadow rules
- `references/composition.md` — section structure, grids, rhythm, and hierarchy
- `references/page-structure.md` — page order and section intent
- `references/architecture.md` — app shell and layout architecture
- `references/components.md` — implementation-oriented component guidance
- `references/motion.md` — reveal, scan, and state motion
- `references/imagery.md` — diagram, thumbnail, and visual treatment rules
- `references/voice.md` — copy tone and vocabulary
- `references/platform-mapping.md` — Tailwind, runtime dependencies, and export mapping

## how to use it

Use this when the direction needs to feel technical, operational, and precise without default SaaS softness. It is a good reference for compliance, infrastructure, analytics, security, and other high-trust product marketing.

With waveframe, this is closest to a handoff-ready extracted system. The point is to keep implementation and design intent aligned: tokens map to code, components map to real sections, and handoff surfaces visualize the system instead of replacing it.

## important note

Some references describe implementation dependencies and runtime files from the original source. Treat those as examples of the right level of specificity. Rename and remap them when applying the system to a different project.

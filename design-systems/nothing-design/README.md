# nothing-design

`nothing-design` is an opt-in reference system for monochrome, industrial, typography-led interfaces.

Use it when someone explicitly asks for a Nothing-inspired direction, a technical monochrome UI, or a system with Swiss structure, OLED black, dot-matrix moments, and mechanical controls.

## what it contains

- `SKILL.md` — operating brief, philosophy, composition rules, anti-patterns, and reference map
- `references/tokens.md` — fonts, type scale, color, spacing, radius, and iconography
- `references/components.md` — cards, buttons, inputs, tables, nav, tags, widgets, overlays
- `references/primitives.md` — page-level layout and section recipes
- `references/animation.md` — motion tokens, named animations, and reduced-motion behavior
- `references/page-moments.md` — loading, empty, error, and success states
- `references/responsive.md` — breakpoints, touch behavior, and escalation rules
- `references/platform-mapping.md` — CSS, font loading, Tailwind, React Native, and SwiftUI mapping
- `references/imagery.md`, `references/logo.md`, `references/voice.md` — brand expression and content rules

## how to use it

Do not apply it automatically. Start with it only when the project direction calls for this visual language.

With waveframe, use it as a quality reference for how a design-system package should be organized: a concise brief, deeper reference files, clear implementation rules, and enough specificity for a future builder or agent to continue the work.

## output expectation

When this system is applied to a project, the useful output is not just a styled screen. Capture the resulting decisions in project-local docs: tokens, components, platform mapping, responsive behavior, and any deviations from the reference system.

# Platform Mapping

Status: **ready for the current runtime**.

This file maps design-system rules to actual runtime files. When any mapped value changes in code, update this file in the same change.

## Framework and Tooling

- Next.js 16 App Router.
- React 19.
- TypeScript.
- Tailwind v4 through `@import "tailwindcss"` and `@theme inline` in `app/globals.css`.
- Bun for scripts/package management.
- Velite for MDX content generation.
- `@paper-design/shaders-react` for procedural paper texture.
- Google Analytics mounted sitewide through `components/google-analytics.tsx` with Google consent mode defaults tied to `report-cookie-consent`. Keep project-specific measurement IDs in environment/config, not reusable design-system docs.
- PostHog provider mounted but inert without env and gated behind accepted cookie consent.
- Resend email stub exists, but forms are not active in this mapping.

## Fonts

Defined in `app/layout.tsx`.

- `JetBrains_Mono` from `next/font/google`, exposed as `--font-jetbrains-mono`.
- `Satoshi` local font files in `app/fonts/`, exposed as `--font-satoshi`.

Runtime truth:

- `--font-sans` resolves to Satoshi.
- `--font-mono` resolves to JetBrains Mono.
- `--font-display` resolves to JetBrains Mono.

The Figma deck displays JetBrains Mono in body slots because Figma cannot load the local Satoshi font files. The runtime contract is still Satoshi sans + JetBrains Mono mono/display, and this file is authoritative.

## Root Layout Mount Order

`app/layout.tsx` mounts:

1. `<GoogleAnalytics />`
2. `<PaperBackground />`
3. `<AnalyticsProvider>`
4. `<Shell>{children}</Shell>`
5. `<FirstVisitLoader />`
6. `<PasswordGate />` unless development bypass is active
7. `<CookieNotice />`

Root metadata:

- `metadataBase`: project domain from `NEXT_PUBLIC_SITE_URL`.
- Default title: `the organization | Digital Asset Expertise`
- Title template: `%s | the organization`
- Default description: `Digital asset investigation, compliance, and training built by practitioners.`
- Canonical, Open Graph, Twitter, and robots defaults are set in `app/layout.tsx`; page-specific metadata comes from `createPageMetadata` in `lib/seo.ts`.
- Apple touch icon: generated by `app/apple-icon.tsx` at `180x180` PNG, based on `/brand/report-ios-save-icon.svg`.

Structured data:

- `components/structured-data.tsx` renders sitewide JSON-LD in the root body.
- Schema graph includes `Organization` and `WebSite`.
- Organization identity uses the approved project name, legal name, logo, and knows-about terms for investigations, blockchain analytics, compliance, and the training institute.

Machine-readable files:

- `/robots.txt` is generated by `app/robots.ts`.
- `/sitemap.xml` is generated by `app/sitemap.ts` from `publicRoutes` in `lib/seo.ts`.
- `/llms.txt` is a static markdown file in `public/llms.txt`.

Analytics consent:

- `components/google-analytics.tsx` installs the Google tag with default consent derived from `localStorage["report-cookie-consent"]`.
- Missing or declined consent maps to denied Google consent mode values.
- Accepted consent maps to granted Google consent mode values and enables PostHog initialization.
- `components/cookie-notice.tsx` persists the choice and updates Google consent on accept/decline.

`<body>` classes:

```txt
relative isolate min-h-full flex flex-col
```

`<body>` data attribute:

```txt
data-wave-signature="built by a wave in progress. waves don't die."
```

## Shell Geometry

Runtime: `components/shell.tsx`.

- Shell wrapper: `relative z-10 mx-auto hidden w-full max-w-[1440px] flex-1 flex-col p-0 md:flex`.
- Frame surface: `bg-surface-page`.
- Frame outline: four absolute `bg-line-strong` lines.
- Gutter rails: `left/right-4 md:left/right-6 xl:left/right-10`.
- Nav: `sticky top-0 z-40 h-14 bg-surface-inverse`.
- Logo block: `h-16 w-16 md:h-24 md:w-24`.
- Desktop nav starts at `lg`.
- Top nav visible labels are `investigations`, `compliance`, `institute`, `who we serve`, `insights`, and `about`; the Investigations href is `/investigations`.
- Desktop nav links are `text-[12px]` and `whitespace-nowrap` so labels do not wrap inside the fixed-height nav row.
- Mobile menu is visible below `lg`, but only where the shell is visible (`md+`).
- `FrameBody`: `px-7 py-12 md:px-10 md:py-16 xl:px-16`; `fill` adds `md:flex-1`.
- `SectionChrome`: `mx-4 h-9 md:mx-6 xl:mx-10`.
- `SectionGap`: `h-[72px] bg-surface-page`.

## Mobile Boundary

Runtime: `components/mobile-breakpoint-blocker.tsx`.

- Below `768px`, a fixed overlay covers the viewport with "mobile loading".
- The shell wrapper is also hidden below `md`.
- This is current implementation truth. It should be removed only as a dedicated responsive-runtime change with docs updated at the same time.

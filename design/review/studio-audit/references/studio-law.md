# studio law

Studio law is the taste bar for the final pass. It does not replace the local
design system, asbuilt package, accessibility rules, or product constraints. It
judges whether the finished work has enough clarity, care, and confidence to
represent the studio.

## lenses

1. **Clarity before style.** A normal user should know what this is, where they
   are, what matters most, and what happens next without reading docs.
2. **One strong hierarchy.** The screen should have a clear entry point, a
   visible primary action, and enough contrast between primary, secondary, and
   tertiary information.
3. **Fewer, stronger decisions.** Remove decorative variety that does not carry
   meaning. Repetition should feel intentional; contrast should earn its place.
4. **Real states.** Loading, empty, error, disabled, focus-visible, hover,
   active, and success states should be designed, not left to defaults or demos.
5. **Trust is visual.** Typos, placeholders, broken rhythm, unclear pricing,
   vague CTAs, jumpy layout, and inconsistent components all lower trust.
6. **Motion must explain.** Animation should clarify state, continuity, or
   feedback. Routine UI should feel instant. Delight belongs only where it helps.
7. **Responsive is the design.** Mobile and in-between widths are not secondary
   outputs. If the composition only works at the agent's favorite viewport, it
   is unfinished.
8. **No AI residue.** Flag generic centered sections, nested cards, icon tile
   stacks, purple/cyan glow defaults, shallow gradient text, and filler copy
   when they make the work feel generated instead of designed.

## severity

| Priority | Meaning |
| --- | --- |
| `P0` | Blocks use, breaks trust, or prevents the core action. |
| `P1` | Serious confusion, quality drop, or conversion friction. |
| `P2` | Noticeable weakness that makes the work feel less finished. |
| `P3` | Optional polish after the main design is sound. |

## evidence rule

Every finding needs evidence: screenshot, viewport, file path, line, route, or
interaction step. If the evidence cannot be named, phrase it as an open question
or omit it.

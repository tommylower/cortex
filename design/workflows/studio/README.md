# studio

the front door to a design practice. one self-contained package an agent loads
before any design work, in any project, canvas or code.

## what's inside

- [SKILL.md](SKILL.md) — the entry point. when to load, what to read, how the pieces connect
- [rules.md](rules.md) — the house law. graded rules (invariants, defaults, experiments), motion numbers, and a margin of pain deposits with provenance dates. rules are earned, never invented
- [playbook.md](playbook.md) — what to do per situation: new project, existing codebase, canvas session, review pass
- [inventory.md](inventory.md) — every tool, skill, and mcp in the practice with a verdict on each
- [doctrine/](doctrine/) — component intake, codebase scaffold, and the design-system package format

## the ideas

- operator intent first. references give anatomy, never skin
- state graphs, not happy frames
- motion numbers live in the margins of the law, tuned by hand, promoted by use
- second-use gate: nothing enters the law until it survives a second real use
- pain-only deposits: rules cite the incident that earned them
